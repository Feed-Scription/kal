# 子系统设计

## NPC 系统

### 管理对象
6 个核心 NPC + 8 个配角 NPC。每个核心 NPC 是一个独立的状态机。

### 核心属性

| 属性 | 初始值 | 范围 | 影响 |
|------|-------|------|------|
| trust（对玩家的信任） | 各异（Zoe:30, 其余:10-15） | [0, 100] | 对话深度、对质成功率、弧线分叉方向 |
| fear（恐惧值） | 各异（Lily:60, Marcus:20） | [0, 100] | 高 fear 时倾向隐瞒和回避 |
| mood（当前情绪基线） | 50（中性） | [0, 100] | 影响社交媒体发帖风格、日常互动态度 |
| arc_stage（弧线阶段） | 1 | [1, 3] | 决定 NPC 当前行为模式和可触发的骨架节点 |
| arc_branch（弧线分支） | "none" | A/B/C/none | 分叉后锁定，决定 NPC 最终命运 |
| loyalty_target（忠诚对象） | 各异 | 字符串 | Lily→Maya, Marcus→Ethan, Coach→学校, Tyler→自己 |

### 信任值变化规则

| 行为 | 信任变化 | 条件 |
|------|---------|------|
| 正面对话（倾听、关心） | +3~+8 | 取决于 NPC 当前 mood |
| 帮助 NPC | +5~+15 | 取决于帮助的重要性 |
| 分享信息 | +5~+10 | 信息必须对 NPC 有价值 |
| 施压/威胁 | -5~-15 | fear 高的 NPC 受影响更大 |
| 欺骗被识破 | -10~-25 | NPC 的 trust 越高，被骗后下降越多 |
| 背叛（泄露 NPC 秘密） | -30~-50 | 可能直接触发弧线负面分支 |
| 忽视（连续 5 天无互动） | -2/天 | 只对 trust>30 的 NPC 生效 |
| 信任共振（隐形规则） | +5 | 连续 3 天出现在同一地点 |

### NPC 情绪模型

mood 值每天根据以下因素波动：
```
daily_mood = base_mood + player_interaction_effect + world_event_effect + arc_pressure
  where:
    base_mood = 50（中性基线）
    player_interaction_effect = clamp(last_interaction_sentiment × 5, -15, +15)
    world_event_effect = 当日世界事件对该 NPC 的影响（-20 ~ +10）
    arc_pressure = -(arc_stage × 5)  // 弧线越深，基础压力越大
```

mood 影响 AI 生成内容的情绪基调：
- mood > 70：相对放松，愿意闲聊
- mood 40-70：正常状态
- mood 20-40：紧张、易怒或沉默
- mood < 20：危机状态，可能触发特殊事件

### 与其他系统的交互

- → 对质系统：trust 影响说服力计算，arc_stage 决定可触发的对质
- → 手机系统：mood 和 arc_stage 影响社交媒体发帖内容和频率
- → 活世界系统：NPC 间的互动受各自 mood 和 arc_stage 影响
- ← 证据系统：玩家展示证据可能改变 NPC 的 fear 和 trust
- ← 派系系统：派系信任度影响配角 NPC 的信息开放度

### 弧线推进规则

弧线阶段推进条件（两种方式，满足任一即推进）：
1. **玩家触发**：满足骨架节点的条件（trust 阈值 + 特定证据/对话）
2. **时间推进**：到达时间截止日且玩家未触发 → NPC 自动推进到下一阶段的默认状态

弧线分叉在阶段 3 的关键对质中决定，一旦锁定不可更改。

## 证据系统

### 管理对象
三类卡牌：话题卡、态度卡、证据卡。

### 卡牌属性

**话题卡**
| 属性 | 说明 |
|------|------|
| id | 唯一标识 |
| name | 显示名称（如"Maya 的失踪"） |
| line | 关联调查线（A/B/C/cross） |
| depth_required | 需要的调查深度才能获得 |
| related_npcs | 对哪些 NPC 有效 |
| relevance | 对各 NPC 的相关度（0-100） |

**态度卡**
| 属性 | 说明 |
|------|------|
| id | 唯一标识 |
| name | 显示名称（如"共情"、"施压"） |
| base_power | 基础说服力加成 |
| npc_weakness | 对哪些 NPC 是弱点（+25 加成） |
| npc_resistance | 对哪些 NPC 有抗性（-15 减免） |
| suspicion_cost | 使用后增加的怀疑值 |

**证据卡**
| 属性 | 说明 |
|------|------|
| id | 唯一标识 |
| name | 显示名称（如"日记碎片"） |
| line | 关联调查线 |
| power | 证据强度（30/60） |
| consumable | 是否一次性（true） |
| freshness | 新鲜度天数（超过后 power 减半） |
| source | 获取途径 |

### 初始卡牌

游戏开始时玩家拥有：
- 话题卡：[转学生身份]、[Maya 失踪（公开信息）]、[Ethan 坠亡（官方说法）]
- 态度卡：[友善]、[好奇]、[直接]
- 证据卡：无

### 卡牌获取

| 获取途径 | 可获得类型 | 说明 |
|---------|----------|------|
| 地点探索 | 话题卡、证据卡 | 特定地点+特定时间 |
| NPC 对话 | 话题卡 | NPC 透露的信息转化为话题 |
| 对质成功（80+） | 话题卡 | 证据信息转化为新话题 |
| 潜入禁区 | 证据卡 | 高风险高回报 |
| 窃听 | 话题卡 | 未过滤信息，可能含误导 |
| 社交媒体 | 话题卡 | 截图可作为弱证据 |
| 私信记录 | 证据卡（弱） | NPC 可能截图给别人 |
| 派系信任解锁 | 话题卡、证据卡 | 达到信任阈值后获得 |

### 与其他系统的交互

- → 对质系统：卡牌组合决定说服力
- → 调查进度：证据数量推进调查深度
- ← NPC 系统：NPC 信任度影响可获得的证据
- ← 手机系统：社交媒体截图和私信记录可转化为证据

## 对质系统

### 管理对象
对质场景实例。每次对质是一个独立的交互序列。

### 说服力公式

```
persuasion = topic_relevance × attitude_power × (1 + evidence_bonus) × (1 + trust_modifier) × (1 - defense_modifier)

where:
  topic_relevance = 话题卡对该 NPC 的相关度 (0.0 ~ 1.0)
  attitude_power = 态度卡基础说服力 (0.5 ~ 1.5)
    + npc_weakness_bonus (0.25 if 命中弱点)
    - npc_resistance_penalty (0.15 if 遇到抗性)
  evidence_bonus = 证据卡强度 / 100 (0 / 0.3 / 0.6)
    × freshness_modifier (1.0 if 新鲜, 0.5 if 过时)
  trust_modifier = state.trust_[npc] / 200 (0.0 ~ 0.5)
  defense_modifier = 0.2 if NPC 对该话题有防备, else 0

final_score = clamp(persuasion × 100, 0, 100)
```

### 结果映射

| 分数 | 结果 | NPC 反应 | 后续影响 |
|------|------|---------|---------|
| 0-29 | 拒绝 | NPC 关闭对话，可能敌对 | trust -10, suspicion +5 |
| 30-59 | 模糊 | NPC 给出含糊回应 | 获得碎片话题卡, trust ±0 |
| 60-79 | 松口 | NPC 透露关键信息 | 获得关键话题卡, trust +5 |
| 80-100 | 坦白 | NPC 完全坦白 | 获得核心话题卡, 可能触发弧线转折, 证据卡信息→新话题卡 |

### 特殊规则

- **沉默的力量**（隐形规则）：空证据槽 + 共情态度 + trust>50 → 30% 概率直接触发坦白
- **连续施压**：对同一 NPC 连续 2 次使用施压态度 → NPC 进入"防御模式"，所有后续对质 defense_modifier +0.3
- **证据链**：在同一次对质中，如果话题卡和证据卡属于同一调查线 → 额外 +15% 说服力

## 手机系统

### 管理对象
4 个子功能：社交媒体、私信、匿名论坛、匿名行动。

### 社交媒体

每天夜间阶段开始时，AI 为每个活跃 NPC 生成 0-2 条动态。

生成规则：
```
post_probability = base_rate × mood_modifier × arc_modifier
  where:
    base_rate = 0.6（每天 60% 概率发帖）
    mood_modifier = 1.0 + (50 - mood) / 100  // mood 低时发帖概率更高
    arc_modifier = 1.0 + (arc_stage - 1) × 0.3  // 弧线越深越活跃
```

帖子属性：
- content：帖子内容（AI 生成）
- tone：情绪基调（反映 mood）
- hidden_clue：是否包含隐藏线索（10% 概率）
- delete_timer：是否会被删除（某些敏感帖子在 1-3 天后删除）
- comments：评论列表（其他 NPC 的反应）

### 私信

玩家可以主动给 NPC 发私信（消耗 1AP）。

规则：
- NPC 回复速度取决于 trust 和 mood（trust 高→快回复，mood 低→慢回复或不回复）
- 私信内容被 AI 根据 NPC 性格生成
- 风险：NPC 可能将私信截图给其他 NPC（概率 = (100 - trust) / 200）
- 私信记录可以截图作为弱证据卡（power: 30）

### 匿名论坛

混合内容池：70% AI 生成 + 30% 预制。

内容类型：
| 类型 | 占比 | 说明 |
|------|------|------|
| 真线索 | 20% | 指向调查线的有效信息 |
| 红鲱鱼 | 15% | 看似有用但误导的信息 |
| 校园八卦 | 35% | 无关但增加世界感的内容 |
| 加密信息 | 5% | 需要"密钥"证据才能解读（指向第四线） |
| NPC 匿名发帖 | 25% | NPC 用匿名账号发的帖子（可通过语言风格识别） |

### 匿名行动

玩家可以用匿名账号执行高风险操作：

| 行动 | 效果 | 风险 |
|------|------|------|
| 散布信息 | 目标 NPC 的 fear +10~+20 | 被识破概率 20%，识破后 suspicion +25 |
| 施压 NPC | 目标 NPC 可能透露信息 | 被识破概率 30%，识破后 trust -20 |
| 制造传闻 | 影响 NPC 间关系 | 被识破概率 15%，可能引发意外后果 |
| 钓鱼 | 引诱 NPC 暴露信息 | 被识破概率 25%，识破后 NPC 进入防御模式 |

识破概率修正：
```
detect_chance = base_chance × (1 + target_npc.fear / 200) × (1 - player.deceptive_score / 100)
```

### 与其他系统的交互

- → NPC 系统：私信和匿名行动影响 NPC 的 trust、fear、mood
- → 证据系统：社交媒体截图和私信记录可转化为证据卡
- → 调查进度：匿名论坛线索可推进调查
- ← NPC 系统：NPC 的 mood 和 arc_stage 影响社交媒体内容
- ← 活世界系统：世界事件影响论坛热点话题

## 活世界系统

### 管理对象
NPC 间的自主互动和世界状态推进。

### 每日世界推进

每天日间结束时（玩家行动完毕后），运行 world-tick Flow：

1. **NPC 位置更新**：根据时间段和弧线阶段，确定每个 NPC 的位置
2. **NPC 间互动**：AI 根据 NPC 关系和当前状态生成 NPC 间的互动事件
3. **弧线时间推进**：检查是否有 NPC 到达时间截止日，自动推进弧线
4. **传闻生成**：NPC 间互动产生的信息进入传闻池，玩家可通过社交媒体或窃听获取
5. **怀疑值衰减**：suspicion 每天自然 -2（最低到 0）

### NPC 间互动规则

每天生成 1-3 个 NPC 间互动事件：

```
interaction_probability(npc_a, npc_b) =
  base_rate × location_overlap × relationship_strength × arc_tension
  where:
    base_rate = 0.3
    location_overlap = 1.0 if 同一地点, 0.2 if 不同地点
    relationship_strength = 关系网中的连接强度 (0.5 ~ 2.0)
    arc_tension = 1.0 + |arc_stage_a - arc_stage_b| × 0.3  // 弧线阶段差异越大越可能互动
```

互动结果影响双方的 mood 和关系，并可能产生传闻。

### 传闻系统

传闻是玩家获取"不在场信息"的主要渠道：

| 传闻来源 | 可靠度 | 获取方式 |
|---------|-------|---------|
| 直接目击（窃听） | 90% | 在场窃听 |
| 一手传闻（NPC 告诉你） | 70% | NPC 对话 |
| 二手传闻（社交媒体） | 50% | 浏览动态 |
| 匿名论坛 | 30% | 匿名论坛 |

传闻可靠度影响玩家获得的信息准确性——不可靠的传闻可能包含夸大、遗漏或完全错误的细节。

### 与其他系统的交互

- → NPC 系统：世界事件影响 NPC 的 mood 和关系
- → 手机系统：NPC 间互动产生社交媒体内容和论坛话题
- → 证据系统：某些世界事件产生可收集的证据
- ← NPC 系统：NPC 的弧线阶段和情绪影响世界事件
- ← 玩家行动：玩家的调查行为影响 NPC 间的互动模式

## 系统交织总览

### 反馈环路

**正反馈（滚雪球）**：
- 信任→信息→证据→对质成功→更多信任：与 NPC 建立信任后获得更多信息，信息转化为证据，对质成功进一步提升信任
- 调查深度→关联发现→跨线证据→更深调查：深入一条线后发现与其他线的关联，获得跨线证据加速其他线的调查

**负反馈（自平衡）**：
- 激进调查→怀疑值上升→行动受限→调查减速：过于激进的调查会引起注意，限制后续行动空间
- 深入一线→时间消耗→其他线停滞→NPC 弧线自动推进（可能走向负面）：专注一条线意味着其他线的 NPC 在没有你介入的情况下走向默认（通常是负面）结局

### 延迟后果链

1. **匿名行动→3天后→NPC 行为变化→5天后→弧线转折**：Day 10 的匿名施压在 Day 13 改变 NPC 行为，在 Day 15 影响弧线走向
2. **证据收集→对质→NPC 弧线分叉→世界事件→其他 NPC 反应**：跨越 5-8 天的因果链
3. **派系信任积累→信息解锁→调查线突破→关键日触发**：跨越 10+ 天的长期投资

### 跨系统 Trade-off

| Trade-off | 系统 A | 系统 B | 张力 |
|-----------|--------|--------|------|
| 速度 vs 安全 | 调查进度 | 怀疑值 | 快速调查提升怀疑值 |
| 深度 vs 广度 | 单线深入 | 多线并行 | 时间有限，不能兼顾 |
| 信任 vs 信息 | NPC 信任 | 证据收集 | 施压获取信息但损害信任 |
| 公开 vs 隐秘 | 面对面互动 | 匿名行动 | 匿名行动效果强但风险高 |
| 个人 vs 全局 | 单个 NPC 命运 | 整体真相 | 保护某人可能需要隐瞒部分真相 |
