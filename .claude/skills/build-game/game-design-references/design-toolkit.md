# 设计工具箱

遇到具体设计难题时，按需查阅对应章节。从 112 个游戏设计透镜蒸馏而来的可操作框架。

---

## 玩家动机模型

自我决定理论（SDT）的三个核心需求解释了为什么玩家会持续玩一个游戏：

| 需求 | 含义 | 游戏中的体现 |
|------|------|------------|
| 自主感 | "我在做自己的选择" | 多条路径、自由探索、个性化选项 |
| 胜任感 | "我在变强/变好" | 难度递进、技能成长、成就反馈 |
| 关联感 | "我与这个世界有连接" | NPC 关系、角色代入、社区归属 |

**动机 × 类型映射**：

| 游戏类型 | 主要动机 | 次要动机 | 设计重点 |
|---------|---------|---------|---------|
| 模拟经营 | 自主感 + 胜任感 | 关联感 | 决策自由度 + 成长可见性 |
| AI 叙事 | 关联感 + 自主感 | 胜任感 | 角色深度 + 选择影响力 |
| 对话冒险 | 关联感 | 自主感 | NPC 关系深度 |

**诊断用法**：如果玩家觉得"没意思"，检查三个需求是否至少满足了两个。缺哪个补哪个。

## 资源经济设计

### 源与汇法则

每种资源都需要**来源**（source）和**消耗**（sink）。如果只有来源没有消耗，资源会通胀；只有消耗没有来源，玩家会绝望。

设计检查表：
- [ ] 每种资源至少有 2 个来源和 2 个消耗途径
- [ ] 来源和消耗的速率大致平衡（略偏紧缺更好）
- [ ] 玩家能感知到资源的流动方向

### 有意义的稀缺

资源的价值来自稀缺。设计稀缺的方法：

| 方法 | 说明 | 示例 |
|------|------|------|
| 总量上限 | 资源有天花板 | 精力值最高 100 |
| 互斥消耗 | 花 A 就不能花 B | 时间只能用于研究或教学 |
| 时间衰减 | 资源随时间减少 | 每回合精力 -10 |
| 机会成本 | 选择一个就放弃另一个 | 本回合只能做一件事 |

### 反馈时机

| 反馈类型 | 时机 | 作用 | 示例 |
|---------|------|------|------|
| 即时反馈 | 行动后立刻 | 满足感、确认操作有效 | 选择后立刻看到数值变化 |
| 短期反馈 | 1-3 回合后 | 策略验证、调整方向 | 本学期的决策影响期末评估 |
| 长期反馈 | 游戏后期 | 成就感、叙事高潮 | 早期选择影响最终结局 |

**原则**：每个行动至少有即时反馈。短期和长期反馈让玩家感到"我的选择有意义"。

## 平衡与难度

### 难度曲线

理想的难度曲线不是线性上升，而是**锯齿形**：

```
难度
 ▲
 │    ╱╲    ╱╲    ╱╲
 │   ╱  ╲  ╱  ╲  ╱  ╲  ╱ ← 高潮
 │  ╱    ╲╱    ╲╱    ╲╱
 │ ╱
 └──────────────────────→ 时间
   教学  探索  挑战  冲刺
```

- **教学期**（前 1/4）：低难度，让玩家熟悉规则
- **探索期**（中 1/4）：中等难度，玩家尝试不同策略
- **挑战期**（中 1/4）：难度上升，考验玩家积累的技能
- **冲刺期**（后 1/4）：最高难度，走向结局

每个阶段内部也有起伏——紧张之后需要喘息。

### 选择平衡检验

对每个选择点，问三个问题：

1. **有没有明显最优选项？** 如果有，要么削弱它，要么给其他选项加buff
2. **最差选项有多差？** 如果选错的惩罚太重，玩家会害怕选择；太轻则选择无意义
3. **选项之间的差异是否可感知？** 如果两个选项效果差不多，合并为一个

### 数值平衡速查

| 参数 | 建议范围 | 说明 |
|------|---------|------|
| 单次事件效果 | ±5% ~ ±15% 总资源 | 太小无感，太大失控 |
| 好结局阈值 | ~top 20% | 需要持续好决策才能达到 |
| 坏结局阈值 | ~bottom 20% | 需要持续坏决策才会触发 |
| 每回合选项数 | 2-4 个 | 超过 4 个导致决策疲劳 |
| 随机事件占比 | 30-50% 的回合 | 太少则可预测，太多则失控 |
| 关键资源波动 | 单回合不超过 ±25% | 防止一次选择毁掉整局 |

## 叙事设计工具

### 兴趣曲线

好的叙事体验像过山车——有起有伏，整体上升，最后有一个最高点：

| 阶段 | 兴趣水平 | 设计手段 |
|------|---------|---------|
| 开场钩子 | 中高 | 用一个悬念或冲突立刻抓住注意力 |
| 铺垫 | 中低 | 介绍世界和角色，建立期待 |
| 第一个转折 | 高 | 打破预期，提出核心冲突 |
| 发展 | 中，逐渐上升 | 深化冲突，增加复杂度 |
| 高潮 | 最高 | 最关键的选择或最大的挑战 |
| 结局 | 中高 → 收束 | 展示选择的后果，给予满足感 |

**关键**：开场钩子必须在前 30 秒内出现。如果开头太平淡，玩家不会等到精彩的部分。

### 契诃夫之枪

> "如果第一幕墙上挂了一把枪，第三幕它必须开火。"

应用到游戏设计：
- 前期引入的每个元素（NPC、物品、信息、地点）都必须在后期发挥作用
- 如果一个元素对后续剧情没有影响，删掉它
- 反过来：后期的关键转折必须在前期有铺垫

### 三的法则

人类对"三"有天然的认知偏好：
- **三个选项**比两个有趣（不是非此即彼），比四个好记
- **三幕结构**（开端-发展-结局）是最基本的叙事框架
- **三次重复**建立模式，第三次打破模式制造惊喜

## 常见设计陷阱速查表

| # | 陷阱 | 症状 | 诊断 | 处方 |
|---|------|------|------|------|
| 1 | 选择幻觉 | 玩家觉得选什么都一样 | 检查选项是否真的改变了游戏状态 | 每个选项至少改变一个状态值 |
| 2 | 数值麻木 | 玩家不关心 +5/-5 | 数值变化缺乏叙事包装 | 用故事描述后果，不只是数字 |
| 3 | 决策疲劳 | 玩家随便选、不想思考 | 选择太频繁或选项太多 | 减少选择频率，每次 2-3 个选项 |
| 4 | 信息过载 | 玩家不知道该关注什么 | 同时展示太多指标/信息 | 可见指标 ≤ 4，其余隐藏或按需展示 |
| 5 | 前期空白 | 开局无聊、没有动力 | 缺少开场钩子 | 第一回合就抛出一个有趣的两难选择 |
| 6 | 后期通胀 | 资源溢出、决策无意义 | 后期缺少资源消耗 | 引入新的消耗维度或提高消耗速率 |
| 7 | 最优路径 | 每次玩都走同一条路 | 缺少随机性或路径间差异不够 | 加入条件事件，让不同状态触发不同内容 |
| 8 | 结局突兀 | 结局感觉莫名其妙 | 结局条件与游戏过程脱节 | 结局必须由玩家的核心决策驱动 |
| 9 | 测评表退化 | 游戏像心理测评：重复"读描述→选选项→看数值变化" | 核心循环只有一种交互形式（选择题），选择后果仅体现为数值变化 | 核心循环包含 ≥2 种交互形式；选择后果必须影响后续可达场景/NPC 态度/可用行动，而非仅改变数值。设计文档中列出的元素（NPC、对话等）必须出现在核心循环描述中 |

## Interlocking Systems Design

好的游戏丰富度不来自"更多内容"，而来自**系统交织产生的涌现行为**——简单规则组合出无穷策略变化。

### System Web Method

列出所有游戏系统。对每对 (A, B)，检查：A 的输出是否影响 B 的输入？B 是否反过来影响 A？

设计目标：

| 目标 | 最低要求 |
|------|---------|
| 每系统连接数 | ≥2 个其他系统 |
| 反馈环路 | ≥1 正反馈（滚雪球）+ ≥1 负反馈（自平衡） |
| 延迟后果链 | ≥1 条跨越 3+ 回合 |
| 跨系统 trade-off | ≥2 个 |

**四种反馈类型：**

| 类型 | 模式 | 效果 | 示例 |
|------|------|------|------|
| 正反馈（滚雪球） | A↑ → B↑ → A↑ | 优势加速 | 更多学生 → 更多论文 → 更高声望 → 更容易招生 |
| 负反馈（自平衡） | A↑ → B↑ → A↓ | 自我纠正 | 更多学生 → 单人关注度下降 → 人均产出降低 |
| 延迟反馈 | A now → B in N 回合 | 期待感 + 规划 | Q1 申请基金 → Q3 出结果 |
| 跨系统 trade-off | A↑ → B↓ | 张力产生 | 横向课题 → 经费↑ 但学生压力↑ |

**检查清单：**
- [ ] 每个系统连接 ≥2 个其他系统
- [ ] 存在 ≥1 正反馈和 ≥1 负反馈环路
- [ ] 存在 ≥1 延迟后果（≥3 回合）
- [ ] 不存在 0 连接的"孤岛"系统

### Emergence from Simple Rules

涌现 = 简单规则 + 共享状态 + 随机方差 → 不可预测的结果。

1. 每条规则独立可理解（一句话能解释）
2. 规则通过共享状态交互（多条规则读写同一数据）
3. 加入随机方差防止确定性路径
4. **测试**：如果你能在未游玩的情况下预测每个结果，说明交互不够

## Anti-Monotony Design

**根因**：单调 = 可预测 + 重复 + 递减的紧迫感。

五个设计支柱对抗后期疲劳：

### Pillar 1: Phase-Gated Mechanics（阶段解锁）

新机制在进程里程碑处解锁。游戏 75% 处的玩法应该和 25% 处明显不同。

| 模式 | 描述 |
|------|------|
| 新系统解锁 | 里程碑后开放全新游戏系统 |
| 机制进化 | 已有机制获得新选项/新深度 |
| 角色转型 | 玩家的能力/身份发生质变 |

**法则：每 25% 进程至少引入 ≥1 个新机制或机制变体。**

### Pillar 2: Tiered Content Pools（分层内容池）

绝不在整个游戏中使用同一个扁平事件池。

```
Phase 1 (0-25%):  单因素事件，温和节奏，引导性
Phase 2 (25-50%): 多因素事件，风险上升，≥2 系统参与
Phase 3 (50-75%): 复合事件（2-3 个同时危机），系统耦合
Phase 4 (75-100%): 级联事件（选择触发延迟后果），最高风险
```

**后期事件应引用前期不存在的系统。** 例如：前期事件只影响个人属性，后期事件同时影响团队、经费和声望。

### Pillar 3: Strategic Pivot Points（策略转换点）

防止"游戏已被解决"——迫使玩家适应新局面。

| 技术 | 描述 |
|------|------|
| 递减收益 | 重复使用同一策略，效果逐次衰减 |
| 资源相变 | 前期充裕的资源在中后期变得稀缺（或反之） |
| 新瓶颈出现 | 新约束使旧策略失效 |
| 目标进化 | 胜利条件在阶段转换时改变或扩展 |

**检验**：玩家能否每回合用完全相同的策略赢得游戏？如果能，缺少转换点。

### Pillar 4: Escalating Interconnection（渐进耦合）

```
前期:  A → B           (简单因果)
中期:  A → B → C       (链式反应)
后期:  A → B → C ↩ A   (反馈环激活，复合涟漪效应)
```

不要一开始就展示全部复杂度。逐步引入系统间耦合，让玩家的认知负荷随技能增长而增长。

### Pillar 5: Discovery Reserves（发现储备）

为坚持到后期和反复游玩的玩家保留惊喜：

| 层级 | 发现时机 | 内容占比建议 |
|------|---------|------------|
| 表层 | 任何一局，前期 | ~50% |
| 隐藏 | 单局后期（条件触发） | ~25% |
| 深层 | 多周目（罕见条件组合） | ~15% |
| 元层 | 社区/wiki 发现 | ~10% |

**法则：≥20% 的总内容在单次通关中不可见。**

### Anti-Monotony Checklist

| # | 检查项 | 通过标准 |
|---|-------|---------|
| 1 | 阶段解锁 | 每 25% 进程有 ≥1 新机制/变体 |
| 2 | 内容分层 | 后期事件池与前期不同，且复杂度更高 |
| 3 | 策略转换 | 存在 ≥1 节点使前期最优策略不再最优 |
| 4 | 渐进耦合 | 后期决策涟漪至 ≥3 个系统 |
| 5 | 隐藏内容 | ≥20% 内容需特定条件才能体验 |

## Numerical Design Framework

### Distribution-Based Attributes

生成实体属性时使用分布，而非固定值：

```
attribute = clamp(N(μ, σ), min, max)
```

| 参数 | 用途 | 设计指引 |
|------|------|---------|
| μ (均值) | 典型值 | 按游戏阶段/实体等级设定 |
| σ (标准差) | 变异度 | 范围的 10-20% 产生适度变异 |
| min/max | 边界钳制 | 防止荒谬极端值 |

**分级分布**：实体质量随上下文变化时（如学校等级影响学生天赋）：
```
μ = baseValue - (tier × step) + contextBonus
σ = baseStd + (tier × varianceGrowth)
```

### Multi-Factor Formulas

关键结果应由 ≥3 因子决定——玩家无法只优化单一维度。

```
outcome = (factorA × wA + factorB × wB) × multiplier1 × multiplier2 × baseRate
```

| 组成 | 作用 | 调参指引 |
|------|------|---------|
| Weights (wA, wB) | 因子相对重要性 | 最大权重 = 玩家最大操控杠杆 |
| Multipliers | 条件修正（1.0=中性） | 特殊状态、奖惩 |
| Base rate | 总体速度/幅度控制 | 先调这个来控制节奏 |
| Clamp | 结果边界 | **必须**对最终结果钳制到合理范围 |

### Diminishing Returns

防止任何单一策略无限堆叠优势：

| 曲线 | 模式 | 场景 |
|------|------|------|
| 软上限 | `bonus = log(1 + value)` | 无硬上限的属性增长 |
| 递增成本 | `cost = base × (level + 1)` | 升级越来越贵 |
| 衰减 | `return = base × (0.9 ^ repeats)` | 重复动作效果递减 |
| 硬上限 | `clamp(value, min, max)` | 绝对边界 |

### Probability Design

| 模式 | 公式 | 场景 |
|------|------|------|
| 基础 + 修正 | `clamp(base + mod1 + mod2, floor, ceiling)` | 成功率 |
| 分级概率 | `if (condA) p1; else if (condB) p2; else pBase` | 稀有事件 |
| 加权随机 | `pick(items, weights)` | 事件/掉落选择 |
| 高斯随机 | `N(μ, σ)` | 结果的自然波动 |

## Replayability Design

### Variable Opening States

每局开始应有所不同：

| 技术 | 描述 |
|------|------|
| 随机初始属性 | 受限分布生成，每局面临不同约束 |
| 开局分支选择 | 早期选择锁定不同策略路线 |
| 条件性内容 | 部分机制仅在特定初始条件下可用 |
| 种子事件 | 前几个事件从池中随机抽取，非固定 |

### Strategic Diversity Test

**问**："是否存在 ≥3 条根本不同的获胜策略？"

如果不是，增加：
- 奖励不同玩法风格的资源路径
- 互斥选择（选 A 永久锁闭 B）
- 多种胜利条件，各有不同要求

### Hidden Content Layers

奖励多周目玩家的内容：
- 需要罕见条件组合的秘密结局
- 绑定特定状态的彩蛋（如：特定职业 + 极端属性 = 隐藏故事线）
- 成就系统逐步揭示游戏深度
- 由前期选择级联触发的后期独占内容

## Multi-Layer Interaction Design

丰富的游戏在多个认知层级提供交互：

| 层级 | 认知负荷 | 示例 |
|------|---------|------|
| 简单动作 | 低（一键） | 散步、休息、使用物品 |
| 战术决策 | 中（权衡 2-3 选项） | 鞭策 vs 安慰、事件选择 |
| 战略规划 | 高（长期规划） | 职业路线、升级投资序列 |
| 社交/情感 | 情感参与（关系经营） | 多轮对话、说服、忠诚/背叛 |
| 发现/探索 | 好奇心驱动 | 隐藏机制、彩蛋、稀有事件 |

**目标：**
- 合格游戏：≥2 层
- 丰富游戏：≥3 层
- 深度可反复游玩的游戏：≥4 层

**深度 > 广度：** 3 个有多因子结果的交互 > 10 个一键操作。每个交互应有：输入决策 → 输出变异 → 级联后果。

## AI 游戏五大死法

AI 游戏特有的反模式。当游戏"结构正确但没人想玩第二遍"时，对照诊断：

| 反模式 | 表现 | 根因 | 修复方向 |
|-------|------|------|---------|
| **问答秀** | 读文字 → 选选项 → 看数字 → 重复 | 只有一种交互模式，没有人的元素 | 加关系/角色，变化交互类型，设计微弧线 |
| **随机生成器** | AI 产出多样但无意义的内容 | AI 在 1 级（内容多样性），没有记忆或叙事穿针 | 加记忆回调、叙事穿针、NPC 声线进化 |
| **电子表格** | 优化数字是唯一的游戏 | 机制没有情感赌注 | 加身份定义时刻、具名赌注、终局之镜 |
| **小说** | 写作优美，没有有意义的能动性 | 选择实际上不重要 | 加不可逆分叉、隐藏系统、跨系统惊喜 |
| **跑步机** | 第 1 回合和第 15 回合感觉一样 | 没有阶段门控，没有发现层 | 设计回合节奏变化，分层系统揭示，门控内容 |

**与"测评表退化"的关系**：问答秀是测评表退化的 AI 游戏特化版本。测评表退化关注交互形式单一，问答秀还叠加了缺乏情感元素的问题。

详细的五大支柱修复框架见 [soul-design-guide.md](soul-design-guide.md)。
