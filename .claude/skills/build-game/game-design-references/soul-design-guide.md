# 游戏灵魂设计指南

Phase 0c/0d 深化设计时加载。诊断和提升游戏的情感吸引力、AI 原生体验和长期可玩性。

**核心问题**：大多数 AI 生成的游戏设计**结构完整但情感空洞**——资源平衡、选择公平、难度合理，但没人想玩第二遍。如果你能把游戏描述为"每回合从选项中选一个，改变一些数字，看结果"——这是结构问题，不是内容问题。更多事件修不好它。

---

## 五大支柱

### 支柱 1：情感锚点 — "我为什么要在乎？"

玩家投入的是*感情*，不是系统。每个好游戏都有一个让玩家*个人投入*的情感核心。

**检验**：如果去掉所有机制，只描述情境，玩家能感受到什么吗？

技巧：
- **具名赌注**：不是"声望 -10"而是"你带了三年的学生要退学了"。数字不产生赌注，关系和身份才会
- **30 秒开场钩子**：第一个交互必须是真正的两难或好奇心触发。不是"欢迎，这是你的属性"而是"你的第一个博士生刚交了论文——署了你对手的名字"
- **情感词汇**：用有情感重量的名字。"理智"比"心理健康分数"有冲击力。"嫌疑"比"风险等级"更有画面感
- **道德时刻**：每局至少设计一个让玩家停下来*思考*的时刻——不是因为数学难，而是因为*道德*难

### 支柱 2：AI 原生魔法 — "只有 AI 才能做到什么？"

大多数"AI 游戏"只是用 AI 替换模板事件。真正的 AI 原生设计创造**没有 AI 就不可能的体验**。

**检验**：如果把所有 AI 调用换成预写内容，游戏会*根本不同*——还是只是少了些变化？

AI 原生体验光谱（目标 3 级以上）：

| 级别 | AI 角色 | 示例 | 玩家感受 |
|------|--------|------|---------|
| 1 | 内容多样性 | 随机 AI 生成事件 | "这是新的" |
| 2 | 情境感知 | 事件引用你的具体状况 | "这跟我有关" |
| 3 | 记忆与连续性 | NPC 记住并引用过去的互动 | "他们认识我" |
| 4 | 涌现人格 | 角色通过游玩发展出独特声音 | "他们是真的" |
| 5 | 共同创作 | 玩家和 AI 创造双方都没计划的故事 | "这是我们一起写的" |

技巧：
- **回调设计**：AI 在新内容中引用玩家过去的决策。"记得你选择资助海外交流吗？那个学生刚在 Nature 发了文章。"——这是最强大的 AI 原生技巧
- **欲望路径**：不要把 AI 限制在预设交互类型中。让玩家尝试意外的方式，AI 有意义地回应。最难忘的时刻来自设计者没预料到的事
- **声线进化**：NPC 的说话方式应随关系历史变化。被严格要求的学生和被呵护的学生说话应该不同——而且应该*随时间变化*
- **叙事穿针**：AI 应在回合间编织连贯故事，不是生成孤立片段。每回合的叙事应建立在前一回合之上，创造玩家能感受到的因果

### 支柱 3：瞬间质感 — "每个回合都令人满足吗？"

宏观设计（核心循环、资源曲线）得到所有关注。但玩家*逐个瞬间*体验游戏。文字游戏的"手感"是：每个单独的交互感觉好不好？

**检验**：即使不关心整体结果，单个回合本身是否令人享受？

技巧：
- **回合内戏剧节奏**：每回合有微弧线：铺垫 → 张力 → 解决 → 下回合钩子。不是"这是你的数字，这是事件，选一个"，而是"学期进展顺利……直到院长召开紧急会议。[选择] → 尘埃落定，但你注意到你最好的学生最近异常安静……"
- **妙语与声音**：游戏的写作应有独特个性。干燥幽默、戏剧反讽或诗意观察。玩家记住的是*怎么说的*，不只是发生了什么
- **情境中的惊喜**：最好的时刻是出人意料*却合理*的。一个连接两个看似无关的早期决策的事件。一个反应出乎意料但完全符合性格的角色
- **关注的奖励**：隐藏细心玩家才能注意到的细节。暗示隐藏系统的属性变化。预示未来事件的 NPC 评论
- **节奏呼吸**：不是每回合都要危机。交替强度：危机 → 安静时刻 → 紧张积累 → 释放。安静回合（散步、学生的小成功）让戏剧回合更有冲击力

### 支柱 4：玩家身份 — "我在这个游戏里是谁？"

好游戏不只让你做选择——它帮你发现*你是什么类型的玩家*。然后回应那个身份。

**检验**：5 回合后，两个玩家能描述完全不同的角色身份吗？

技巧：
- **玩法风格识别**：追踪的不只是结果而是*模式*。这个玩家总选冒险？总保护学生？总优化金钱？然后让游戏世界*回应*这个模式——NPC 评论它，事件引用它，AI 叙述者的语气也随之变化
- **有意义的不可逆**：每局至少 2-3 个选择永久关闭选项同时打开其他选项。"你选择专攻 AI 研究"应该有意义地改变游戏的可能性空间，不只是翻转一个标记
- **身份定义时刻**：设计至少一个玩家必须在两个一直在建设的东西之间选择的时刻——为学生的福祉牺牲研究，或明知有人受伤仍然推进
- **终局之镜**：游戏结束时（或关键里程碑），以有洞察力的方式反映玩家的选择。不是"你选了 A 12 次"而是"你建了一个忠诚比产出更重要的实验室。学生们爱你——虽然评审委员会有保留意见。"

### 支柱 5：发现架构 — "还有什么是隐藏的？"

长期参与来自感觉总有更多东西可以发现。不只是隐藏内容，还有隐藏的*理解*。

**检验**：完成一次游戏后，玩家是否怀疑有整个系统或故事他们没见过？

技巧：
- **分层系统**：一些游戏规则应该不可见，只能通过游玩发现。"等等，在第一年给学生送礼有复利效果？"这奖励实验并创造讨论话题
- **跨系统惊喜**：当两个系统以感觉涌现（非设计）的方式交互时，产生愉悦。刻意设计这些交叉但让它们感觉偶然
- **条件独占内容**：至少 20% 的内容在单次通关中不可达。不是通过随机，而是通过*选择后果*——走研究路线的玩家看到完全不同的后期事件
- **元理解递进**：每次通关教会玩家关于*游戏如何运作*的新东西。第一局：学基础。第二局：发现隐藏交互。第三局：尝试高级策略
- **低语系统**：偶尔暗示玩家没找到的内容。NPC 提到玩家没触发的事件。指标短暂出现又消失

## 灵魂审查清单

设计蓝图完成后（Phase 0d）或游戏感觉无生气时使用：

| # | 检查项 | 失败信号 | 修复方向 |
|---|-------|---------|---------|
| 1 | **情感锚点**存在 | 核心循环描述中没有人的元素 | 加入玩家在乎失去的角色、关系或身份 |
| 2 | **开场钩子**在第一个交互中 | 游戏以属性/教程开始 | 以两难、谜题或戏剧情境开场 |
| 3 | **AI 达到 3 级+** | AI 只生成"新事件" | 加入记忆回调、声线进化、叙事穿针 |
| 4 | **回合级节奏**已设计 | 每回合结构相同 | 变化回合强度，加入安静时刻，设计微弧线 |
| 5 | **独特声音** | 叙事功能性/通用 | 在 AI prompt 中定义语气，加入妙语/个性/风格 |
| 6 | **玩家身份**被识别 | 游戏对所有玩法风格一视同仁 | 追踪模式，让世界回应玩家的"类型" |
| 7 | **不可逆选择**存在（≥2） | 所有选择可恢复/渐进 | 加入永久重塑游戏的分叉点 |
| 8 | **隐藏系统**奖励发现 | 一切都提前解释 | 加入玩家通过实验发现的隐形规则 |
| 9 | **跨系统惊喜**已设计 | 系统独立运作 | 设计 3+ 个系统意外交互的"交叉时刻" |
| 10 | **终局之镜**反映旅程 | 结局只是属性阈值 | 生成解读玩家旅程的叙事 |
| 11 | **安静时刻**存在 | 每回合都高强度 | 设计 20-30% 的回合为有微妙回报的呼吸空间 |
| 12 | **好奇心缺口**驱动重玩 | 重玩只改变随机结果 | 在选择独占路径后隐藏 20%+ 内容 |

**评分**：每项 1 分。8+ 分 = 有吸引力的游戏。低于 6 分 = 结构完整但感觉像电子表格。

## AI 游戏五大死法（反模式）

| 反模式 | 表现 | 根因 | 修复 |
|-------|------|------|------|
| **问答秀** | 读文字 → 选选项 → 看数字 → 重复 | 只有一种交互模式，没有人的元素 | 支柱 1+3：加关系，变化交互类型，设计微弧线 |
| **随机生成器** | AI 产出多样但无意义的内容 | AI 在 1 级，没有记忆或穿针 | 支柱 2：加回调、叙事穿针、声线进化 |
| **电子表格** | 优化数字是唯一的游戏 | 机制没有情感赌注 | 支柱 1+4：加身份定义时刻、个人赌注、终局之镜 |
| **小说** | 写作优美，没有有意义的能动性 | 选择实际上不重要 | 支柱 4+5：加不可逆分叉、隐藏系统、跨系统惊喜 |
| **跑步机** | 第 1 回合和第 15 回合感觉一样 | 没有阶段门控，没有发现层 | 支柱 3+5：设计回合节奏，分层系统揭示，门控内容 |

## 实现模式与 Prompt 模板

Phase 4（实现 Flow）和 Phase 6（内容填充）时参考。

### 具名赌注（支柱 1）

把数字变成人和故事。游戏中每个关键指标至少关联一个具体角色或事物。

**Before**: `研究值 +5, 经费 -10000, 声望 +3`
**After**: `小张的论文终于发表了——你通宵帮她改了三遍。经费吃紧，但看到她发朋友圈时的笑脸，你觉得值了。[声望 +3]`

### 滴答关系（支柱 1）

在背景中维护一个缓慢发展的关系线。玩家不需要每回合交互，但这个关系在暗中演变——直到某一刻爆发，变成关键决策点。

实现：在 state 中维护角色关系历史 key + AI 在 PromptBuild 的 `field` fragment 中注入该角色的历史互动。

### 记忆回调（支柱 2）

**Prompt 模板**:
```
你正在为一款模拟游戏生成本回合事件。

当前状态: {compressed_state}

关键历史决策（必须在事件中引用至少 1 个）:
{key_decisions}

生成一个与当前状态相关的事件，其中至少一个选项的后果
与上述某个历史决策产生因果关联。用"因为你之前..."或
类似表述建立因果。
```

实现关键：在 state 的 history 数组中标记 "key decisions"（影响大于阈值的决策），通过 PromptBuild 的 `field` fragment 只传这些关键决策给 LLM。

### NPC 声线进化（支柱 2）

**Prompt 模板**:
```
你在扮演 {name}。性格: {personality}。

与玩家的关系历史:
- 关系等级: {level} (陌生/熟悉/信任/亲密)
- 近期互动: {recent_interactions}
- 情绪基线: {current_mood}

语调规则:
- 陌生: 礼貌、正式、保持距离，用"您"
- 熟悉: 放松但尊重，偶尔开小玩笑
- 信任: 坦诚、有时直接表达不满或关心
- 亲密: 像朋友一样说话，会主动分享私事

不要在对话中暴露你知道自己的"关系等级"。自然地表现出这个阶段的语气。
```

### 叙事穿针（支柱 2）

**Prompt 模板**:
```
生成本回合的叙事总结。必须包含三个部分：

1. 【回响】一句话提及上回合的关键事件及其余波
2. 【当下】本回合发生的主要事情（2-3句）
3. 【暗流】一句话暗示下回合可能出现的问题或机遇（要微妙，不要直接剧透）

上回合关键事件: {last_event}
本回合模拟结果: {current_result}
潜在下回合触发器: {next_hints}
```

### 欲望路径支持（支柱 2）

在对话系统中，除了预设选项，允许玩家自由输入：

```typescript
choices: [
  { id: 'comfort', label: '安慰她' },
  { id: 'push', label: '鞭策她' },
  { id: 'free', label: '说点别的...', freeInput: true },
]
```

### 微弧回合（支柱 3）

**三拍结构**:
```
拍 1（常态）: "这个月的实验室生活平静如常，小张在赶论文，老王在带新生..."
拍 2（冲突）: "直到周三下午——小张发现她的实验数据和隔壁课题组的论文高度相似。"
拍 3（抉择）: [选项呈现，每个都有不同的风险和情感重量]
```

**AI Prompt 要求**:
```
事件必须遵循三拍结构:
1. 用一句话建立"一切正常"的基调
2. 用"直到/然而/但是"引入转折
3. 以需要决策的困境结束
```

### 妙语与个性（支柱 3）

给游戏的叙事声音一个明确的人格。在 system prompt 中明确要求：

```
你是一个带有干燥幽默感的叙事者。你的叙事风格类似《围城》
——用精准的讽刺描写荒谬的现实，但底色是温暖和同情。
绝不用感叹号。用冷静的语气说最戏剧性的事。
```

**风格示例**：
- **干燥幽默**: "评审意见回来了。好消息：只有 47 条修改意见。坏消息：第一条是'建议投其他期刊'。"
- **黑色喜剧**: "你在咖啡机前遇到了院长。他热情地拍你肩膀，用那种'我要给你派活'的笑容看着你。"
- **冷峻写实**: "基金申请被拒。理由：'该方向已有成熟成果。' 你数了数，这是今年第三封内容几乎相同的拒信。"

### 安静回合（支柱 3）

设计 20-30% 的回合为"呼吸"回合：没有重大决策，但有小而温暖的细节；暗中推进长期事件线；给玩家整理思路的空间；通常包含伏笔或隐藏信息。

### 玩家 DNA 追踪（支柱 4）

在 `initial_state.json` 中声明一个 `playstyleTags` 对象类型的 state key，通过自定义 Node 或 `ApplyState` 在每次选择后更新：

```json
"playstyleTags": { "type": "object", "value": { "nurturing": 0, "bold": 0, "strategic": 0 } }
```

在 Flow 中，每次玩家做出选择后，LLM 输出的 `stateChanges` 可以包含 `playstyleTags` 的更新。然后在 `PromptBuild` 的 `field` fragment 中注入 `state.playstyleTags`，让 AI 参考玩家的"性格画像"生成贴合的内容。

传给 AI 时，让它参考玩家的"性格画像"。一个"铁血导师"型玩家应该遇到挑战其铁血的事件；一个"佛系导师"型玩家应该遇到逼他不得不强硬的情况。

### 不可逆分叉（支柱 4）

在游戏 25% 和 50% 处设置至少各一个不可逆选择：
- 两条路径各自有独占内容（≥3 个独占事件）
- 选择后游戏世界可见地改变（UI、可用行动、NPC 态度）
- 不存在"正确"路径——两条路各有独特的好处和代价

### 终局之镜（支柱 4）

**Prompt 模板**:
```
基于这位角色的历程，写一段 300 字的传记式回顾。

要求:
1. 不要列举数据。用故事的方式讲述。
2. 找到这位角色的"核心特质"——TA的选择中有什么一致的模式？
3. 提到至少 2 个具体事件（从历史记录中选取）
4. 最后一句话要有"盖棺定论"的感觉——像一句墓志铭或传记结语
5. 语调要庄重但不煽情

玩家选择倾向: {playstyle_tags}
关键事件: {career_highlights}
最终数据: {final_stats}
结局类型: {ending_type}
```

### 隐形规则（支柱 5）

设计 2-3 个不在说明中展示的规则，玩家只能通过游玩发现：
- 连续 3 回合安慰同一角色 → 解锁"知心"特殊状态，产出 +30%
- 资源低于阈值但声望高于阈值 → 触发隐藏事件线
- 所有角色压力都低于阈值 → 团队氛围加成

关键：效果要足够明显让发现者觉得"原来如此！"，但不要明显到一眼看出。

### 低语系统（支柱 5）

在正常内容中偶尔暗示玩家没有触发的内容：
- NPC 提到玩家没见过的事件
- 日志中出现被忽略的线索
- 某条选择的 tooltip 暗示隐藏影响

### 交叉时刻（支柱 5）

有意设计系统间碰撞出惊喜的时刻：
1. 列出所有系统对 (A, B)
2. 对每一对，问："如果 A 处于极端状态时 B 也处于极端状态，会发生什么有趣的事？"
3. 选择最有戏剧性的 3-5 个组合，设计专属事件

## AI System Prompt 灵魂清单

在写任何 AI system prompt 时，检查是否包含：

- [ ] **语气指令**：明确的叙事风格（幽默/冷峻/温暖/讽刺）
- [ ] **回调指令**："必须引用至少 1 个历史决策"
- [ ] **三拍指令**："事件遵循 常态→转折→抉择 结构"
- [ ] **禁止清单**：不要做什么（不要直接说数值、不要信息量过大、不要用感叹号等）
- [ ] **惊喜空间**："偶尔（10%概率）加入一个意想不到但合理的元素"
- [ ] **情感锚点**："让选择涉及一个具体的人，而非抽象的指标"
