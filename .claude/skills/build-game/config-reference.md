# KAL-AI Project File Reference

> Templates for all configuration files in a KAL-AI game project. Use these when setting up a new game.

---

## Project Directory Structure

```
{game-name}/
├── kal_config.json        # Engine + LLM configuration (required)
├── initial_state.json     # Game state initialization (required)
├── session.json           # Interaction state machine (required for kal play)
├── flow/                  # Flow JSON files (required)
│   ├── main.json
│   ├── intro.json
│   ├── narrate.json
│   └── ...
├── node/                  # Custom node implementations (optional)
│   └── dice-roll.ts
└── design/                # Design documents (generated by Phase 0)
    ├── README.md
    ├── rules.md
    ├── content.md
    └── ...
```

## kal_config.json

```json
{
  "name": "{game-name}",
  "version": "1.0.0",
  "engine": {
    "logLevel": "warn",
    "maxConcurrentFlows": 1,
    "timeout": 60000
  },
  "llm": {
    "provider": "openai",
    "apiKey": "${OPENAI_API_KEY}",
    "baseUrl": "${OPENAI_BASE_URL}",
    "defaultModel": "gpt-4o-mini",
    "retry": {
      "maxRetries": 2,
      "initialDelayMs": 1000,
      "maxDelayMs": 10000,
      "backoffMultiplier": 2,
      "jitter": true
    },
    "cache": {
      "enabled": false
    }
  }
}
```

### Config fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Project name |
| `version` | Yes | Project version |
| `engine.logLevel` | No | `"debug" \| "info" \| "warn" \| "error"` (default: `"warn"`) |
| `engine.maxConcurrentFlows` | No | Max parallel flow executions (default: 1) |
| `engine.timeout` | No | Flow execution timeout in ms (default: 60000) |
| `llm.provider` | Yes | LLM provider (`"openai"` for OpenAI-compatible APIs) |
| `llm.apiKey` | Yes | API key (supports `${ENV_VAR}` syntax) |
| `llm.baseUrl` | No | API base URL (supports `${ENV_VAR}` syntax) |
| `llm.defaultModel` | Yes | Default model name |
| `llm.retry` | No | Retry config with exponential backoff + jitter |
| `llm.cache` | No | In-memory cache config (TTL + maxEntries + LRU) |

**Environment variable substitution**: `${VAR_NAME}` in string values is replaced with the corresponding environment variable at load time. This is handled by `ConfigLoader.parse`.

## initial_state.json

Flat key-value map. Each key is a `{ type, value }` pair.

```json
{
  "playerName": {
    "type": "string",
    "value": "冒险者"
  },
  "race": {
    "type": "string",
    "value": ""
  },
  "class": {
    "type": "string",
    "value": ""
  },
  "background": {
    "type": "string",
    "value": ""
  },
  "health": {
    "type": "number",
    "value": 100
  },
  "maxHealth": {
    "type": "number",
    "value": 100
  },
  "strength": {
    "type": "number",
    "value": 10
  },
  "gold": {
    "type": "number",
    "value": 50
  },
  "currentLocation": {
    "type": "string",
    "value": "镇中心广场"
  },
  "inventory": {
    "type": "array",
    "value": ["铁剑", "火把x3", "面包x2"]
  },
  "skills": {
    "type": "array",
    "value": []
  },
  "questStage": {
    "type": "string",
    "value": "arrived"
  },
  "history": {
    "type": "array",
    "value": []
  },
  "summary": {
    "type": "string",
    "value": ""
  },
  "createMode": {
    "type": "string",
    "value": ""
  }
}
```

### Supported types

| Type | JSON value | Notes |
|------|-----------|-------|
| `string` | `"text"` | Any string |
| `number` | `42` | Must pass `isFinite` check |
| `boolean` | `true` / `false` | Strict boolean |
| `object` | `{ ... }` | Plain JSON object |
| `array` | `[ ... ]` | JSON array |

### Design guidelines

- **Always include `history`** (array) — `GenerateText` auto-appends conversation turns here
- **Always include `summary`** (string) — For history compaction in long games
- **Pre-declare all LLM-writable keys** — `ApplyState` only modifies existing keys
- **Keep it flat** — No nesting inside `value`. Each game variable is a top-level key

## session.json

```json
{
  "schemaVersion": "1.0.0",
  "name": "Game Title",
  "description": "One-line game description",
  "steps": [
    {
      "id": "intro",
      "type": "RunFlow",
      "flowRef": "intro",
      "next": "choose-mode"
    },
    {
      "id": "choose-mode",
      "type": "Choice",
      "promptText": "选择你的角色：",
      "options": [
        { "label": "战士 — 近战专家", "value": "warrior" },
        { "label": "法师 — 魔法大师", "value": "mage" },
        { "label": "自定义创建", "value": "custom" }
      ],
      "flowRef": "apply-preset",
      "inputChannel": "choice",
      "next": "check-mode"
    },
    {
      "id": "check-mode",
      "type": "Branch",
      "conditions": [
        { "when": "state.createMode == 'custom'", "next": "input-name" }
      ],
      "default": "start-adventure"
    },
    {
      "id": "input-name",
      "type": "Prompt",
      "stateKey": "playerName",
      "promptText": "为你的角色取一个名字：",
      "next": "start-adventure"
    },
    {
      "id": "start-adventure",
      "type": "RunFlow",
      "flowRef": "start-adventure",
      "next": "turn"
    },
    {
      "id": "turn",
      "type": "Prompt",
      "flowRef": "main",
      "inputChannel": "playerInput",
      "promptText": "你的行动？",
      "next": "check"
    },
    {
      "id": "check",
      "type": "Branch",
      "conditions": [
        { "when": "state.health <= 0", "next": "death" },
        { "when": "state.questStage == 'completed'", "next": "victory" }
      ],
      "default": "compact-check"
    },
    {
      "id": "compact-check",
      "type": "Branch",
      "conditions": [
        { "when": "state.history.length >= 10", "next": "do-compact" }
      ],
      "default": "turn"
    },
    {
      "id": "do-compact",
      "type": "RunFlow",
      "flowRef": "compact-history",
      "next": "turn"
    },
    {
      "id": "death",
      "type": "RunFlow",
      "flowRef": "outro-death",
      "next": "end"
    },
    {
      "id": "victory",
      "type": "RunFlow",
      "flowRef": "outro-win",
      "next": "end"
    },
    {
      "id": "end",
      "type": "End"
    }
  ]
}
```

### Step type reference

| Type | Required Fields | Optional Fields |
|------|----------------|-----------------|
| `RunFlow` | `id`, `flowRef`, `next` | — |
| `Prompt` | `id`, `promptText`, `next` | `stateKey`, `flowRef`, `inputChannel` |
| `Choice` | `id`, `promptText`, `options[]`, `next` | `stateKey`, `flowRef`, `inputChannel` |
| `Branch` | `id`, `conditions[]`, `default` | — |
| `End` | `id` | — |

**Choice option**: `{ "label": "Display text", "value": "stored_value" }`

**Branch condition**: `{ "when": "state.xxx == 'value'", "next": "step-id" }` — supports `==`, `!=`, `<=`, `>=`, `<`, `>` operators on state values.

### Minimal Flow (static output, no LLM)

```json
{
  "meta": {
    "schemaVersion": "1.0.0",
    "name": "intro",
    "description": "Opening narration",
    "inputs": [],
    "outputs": [{ "name": "result", "type": "string" }]
  },
  "data": {
    "nodes": [
      {
        "id": "prompt-build",
        "type": "PromptBuild",
        "label": "Build text",
        "position": { "x": 0, "y": 0 },

        "outputs": [
          { "name": "messages", "type": "ChatMessage[]" },
          { "name": "text", "type": "string" },
          { "name": "estimatedTokens", "type": "number" }

      {
```

```json
    "nodes": [

        "id": "signal-in",

        "id": "narrate",

        "label": "Narration sub-flow",
```

## Running the Game

### Prerequisites

```bash
# Set LLM environment variables
export OPENAI_API_KEY=your_api_key
export OPENAI_BASE_URL=https://your-openai-compatible-endpoint

# Build the engine (if not already built)
pnpm --filter @kal-ai/engine build
```

### Interactive TUI

```bash
node apps/engine/dist/bin.js play {game-name}
```

TUI commands: `/quit`, `/state`, `/help`

### HTTP API Server

```bash
node apps/engine/dist/bin.js serve {game-name}
# Default: http://localhost:3000
```

### Editor (visual Flow inspector)

```bash
cd apps/editor && pnpm dev
# Connects to Engine at http://localhost:3000
```
