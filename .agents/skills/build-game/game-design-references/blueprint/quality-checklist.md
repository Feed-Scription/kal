# 质量检查清单

输出蓝图前，逐项检查：

| # | 检查项 | 通过标准 |
|---|-------|---------|
| 1 | 一句话描述 | 能让没玩过的人立刻理解这是什么游戏 |
| 2 | 核心循环 | 能清晰描述玩家重复做的事，且循环本身有趣 |
| 3 | 交互多样性 | 核心循环包含至少 2 种不同的交互形式（见 [assessment-trap.md](assessment-trap.md)） |
| 4 | 选择有后果 | 选择产生的影响在后续场景/事件中可见体现，而非仅改变数值 |
| 5 | 可见指标 | 不超过 4 个，每个都影响玩家决策 |
| 6 | 资源源汇 | 每种资源都有来源和消耗，且列出了极端后果 |
| 7 | 选择平衡 | 没有明显最优选项，每个选择都有取舍 |
| 8 | 结局覆盖 | 至少 3 个结局（好/中/坏），有默认兜底结局 |
| 9 | 内容量 | MVP 事件/场景数量足够支撑核心循环（模拟类 ≥15 事件，叙事类 ≥10 场景） |
| 10 | 契诃夫之枪 | 蓝图中提到的每个元素都在流程中发挥作用，没有悬空元素 |
| 11 | 回合数合理 | 总回合数在建议范围内（模拟类 12-24，叙事类 10-20 场景），不会造成疲劳 |
| 12 | 设计说明 | 解释了关键决策的理由，且关联到目标美学 |
| 13 | AI 应用规划 | 明确了哪些系统用 AI、后备内容量、降级策略。游戏无 AI 时仍可完整游玩 |
| 14 | 子系统深度 | 每个核心系统有独立的属性、公式、交互规则定义（Deep tier 必需） |
| 15 | 系统交织 | 系统间存在 ≥2 个反馈环路和 ≥1 个延迟后果链（详见 design-toolkit.md "Interlocking Systems Design"） |
| 16 | 后期新鲜感 | 每 25% 游戏进程至少引入 1 个新机制或机制变体（详见 design-toolkit.md "Anti-Monotony Design"） |
| 17 | 隐藏内容 | ≥20% 内容在单次通关中不可见（隐藏结局、彩蛋、条件触发事件等） |
| 18 | 策略多样性 | 存在 ≥3 种根本不同的获胜策略路线 |
| 19 | 数值设计 | 关键结果由 ≥3 个因子决定，有合理的钳制范围（详见 design-toolkit.md "Numerical Design Framework"） |
| 20 | 灵魂审查 | 通过 [soul-design-guide.md](../soul-design-guide.md) 灵魂审查清单评分 ≥6（Standard）或 ≥8（Deep）。重点检查：情感锚点、AI 3级+、回合节奏、不可逆选择 |

**注**：检查项 14-19 对 Quick tier 为可选，Standard tier 建议满足 ≥3 项，Deep tier 必须全部满足。检查项 20 对 Standard 和 Deep tier 必需。

如有未通过项，回到 Phase 0c 补充对应内容。
