# design/ 文件夹模板

设计输出为一个文件夹，而非单一大文件。每个文件聚焦一个关注点，实现阶段按需加载。

## 文件夹结构

```
design/
├── README.md              # 必须 — 设计定位 + 文件索引
├── rules.md               # 必须 — 规则 + 资源 + 结局
├── content.md             # 必须 — 元素 + 流程
├── ai-plan.md             # 必须 — AI 规划 + 后备需求
├── soul.md                # 必须 — 灵魂设计
├── subsystems.md          # Standard/Deep — 子系统 + 公式
└── progression.md         # Deep — 后期新鲜感 + 隐藏内容
```

---

## README.md（必须）

```markdown
# [游戏名称]

## 设计定位

- **目标美学**：[2-3 个核心体验，如：挑战征服 + 表达创造]

- **引擎映射建议**：[Session 结构、关键 Flow 列表、核心 Node 链路]
- **Design Depth Tier**：[Quick / Standard / Deep]

## 一句话描述

[用一句话说明这是什么游戏，让没玩过的人立刻理解]

## 核心玩法

[用通俗语言描述玩家主要做什么，2-3 段]

## 设计说明

[解释关键设计决策的理由：为什么选择这些资源？为什么这样设计核心循环？这些设计如何服务于目标美学？]

## KAL 实现映射建议

[列出建议的 Session 结构、关键 Flow 列表、核心 Node 链路，供技术实现阶段参考]

## 文件索引

| 文件 | 内容 | 实现阶段参考 |
|------|------|------------|
| rules.md | 游戏规则、资源指标、结局 | Phase 2 (State), Phase 4 (Flows) |
```

## ai-plan.md（必须）

```markdown
# AI 应用规划

AI 是游戏的灵魂——它让每局游戏都有独特的叙事体验。明确规划哪些系统使用 AI、哪些使用确定性逻辑。

## AI 应用范围

| 游戏系统 | AI 模式 | 说明 |
|---------|---------|------|
| [如：回合结算] | [纯数值 / AI叙事+数值 / 纯AI] | [AI 生成叙事日志，规则验证数值] |
| [如：随机事件] | [纯预制 / 混合(推荐) / 纯AI] | [70% AI 生成 + 30% 预制后备] |
| [如：NPC 对话] | [无 / 模板 / AI对话] | [AI 生成个性化回复和选项] |
| [如：角色生成] | [预制 / AI+预制后备] | [AI 生成人格，后备 15+ 模板] |
| [如：结局总结] | [纯数据 / AI叙事] | [AI 生成文学性职业回顾] |

## 后备内容需求

| 内容类型 | 最低数量 | 说明 |
|---------|---------|------|
| [如：预制事件] | [≥15] | [确保无 AI 时不重复] |
| [如：角色模板] | [≥15] | [含反应文本] |
| [如：社交动态] | [≥20] | [混合类型] |

## AI 架构要点

- **核心原则**：AI 生成创意内容 → 游戏规则验证约束 → 应用到状态。永远不让 AI 直接修改游戏状态
- **降级策略**：每个 AI 调用都有 try-catch + 后备数据，游戏无 AI 时仍完整可玩
- **本地模式**：[是否支持纯离线游玩？如何切换？]
```
