## 模拟经营类

### 核心循环

```
每回合：选择行动 → 消耗资源 → 触发事件 → 获得/失去资源 → 时间推进
```

玩家在有限时间内管理多种互相挤压的资源，通过决策优化结果。

### 三资源挤压法则

模拟经营的核心张力来自**资源互斥**——提升 A 必然牺牲 B 或 C。

设计方法：
1. 选择 3-4 种核心资源（如：资金 / 声誉 / 满意度 / 精力）
2. 确保每个行动至少消耗一种资源、产出另一种
3. 没有任何行动能同时提升所有资源——这是产生有意义决策的关键

**检验**：如果存在一个行动总是最优选择（无论什么情况都该选它），说明资源挤压不够。

### 指标设计

三层指标体系让游戏既透明又有深度：

| 层级 | 玩家可见性 | 用途 | 示例 |
|------|-----------|------|------|
| 可见指标 | 始终显示 | 玩家决策的主要依据 | 资金、声誉、满意度 |
| 内部指标 | 玩家可查看但不突出 | 影响事件触发和结局 | 教学质量、科研积累 |
| 隐藏指标 | 完全不可见 | 驱动 AI 生成和规则判定 | 压力值、运气系数 |

**原则**：可见指标不超过 4 个。太多指标会让玩家决策瘫痪。

### 事件设计

| 要素 | 建议值 | 说明 |
|------|-------|------|
| 事件池大小 | 20-40 个 | MVP 可从 15 个起步 |
| 权重分布 | 常见 30-50 / 普通 15-25 / 稀有 5-10 | 稀有事件制造惊喜 |
| 条件事件占比 | 30-50% | 让游戏状态影响事件出现 |
| 每个事件选项数 | 2-3 个 | 超过 3 个会拖慢节奏 |
| 选项效果范围 | ±5% ~ ±15% 的资源变化 | 太小无感，太大失控 |

**事件叙事公式**：情境描述（1-2 句）→ 两难选择 → 每个选项有明确的代价和收益。避免"明显正确答案"的选项。

### 时间节奏

| 要素 | 建议 |
|------|------|
| 总回合数 | 12-24 回合（太短没有成长感，太长会疲劳） |
| 时间单位 | 与主题匹配（学期/季度/月/周） |
| 阶段划分 | 前期探索（1/4）→ 中期发展（1/2）→ 后期冲刺（1/4） |
| 节奏变化 | 每 3-5 回合有一个"里程碑事件"打破常规 |

### 结局设计

| 要素 | 建议 |
|------|------|
| 结局数量 | 3-6 个（1-2 好结局 + 1-2 中性 + 1-2 坏结局） |
| 触发方式 | 指标阈值（如声誉 > 80）或特殊条件组合 |
| 阈值设计 | 好结局 ~top 20%，坏结局 ~bottom 20% |
| 默认结局 | 必须有一个"普通结局"兜底 |

### 常见陷阱

| 陷阱 | 症状 | 处方 |
|------|------|------|
| 资源通胀 | 后期资源溢出，决策无意义 | 增加后期消耗项或引入新资源维度 |
| 最优路径 | 每次玩都走同一条路 | 加入随机事件打破固定策略 |
| 数值麻木 | 玩家不关心 +5/-5 的变化 | 用叙事包装数值变化，让玩家感受到后果 |
| 前期无聊 | 开局选择太少 | 用一个强力开场事件立刻抛出两难选择 |
| 电子表格化 | 游戏只是优化数字 | 每个关键指标关联具体角色（具名赌注），加入不可逆分叉和终局之镜 |

### 灵魂要点（模拟经营类）

模拟经营最容易陷入"电子表格"反模式——玩家只在优化数字。关键对策：
- **具名赌注**：每个核心指标关联至少一个具体角色或事物。不是"教学能力"而是"学生们是否信任你"
- **滴答关系**：在背景中维护一个缓慢发展的关系线，在关键时刻爆发为决策点
- **安静回合**：20-30% 的回合没有重大决策，但有温暖细节和伏笔
- **AI 回调**：AI 生成事件时必须引用至少 1 个历史决策，建立因果感

详见 [soul-design-guide.md](soul-design-guide.md) 支柱 1（情感锚点）和支柱 3（瞬间质感）。

### KAL 实现映射

```
Session Branch     → 回合计数 + 阶段判定（用 state 中的 turn/phase 字段）
State keys         → 三层指标（可见指标直接展示，隐藏指标用于 LLM prompt 和 Branch 条件）
PromptBuild field  → 动态注入当前指标到 LLM prompt
GenerateText       → AI 生成事件叙事 + stateChanges
WriteState         → 回写指标变化（allowedKeys 白名单约束）
Session Branch     → 结局判定（state.health <= 0 等条件）
history array      → 游戏日志（供 LLM 上下文 + 历史压缩）
```

### Depth Scaling（Deep tier 专用）

#### Subsystem Design Checklist

对每个核心系统，定义：
1. **实体与属性**：管理什么对象？每个属性的分布 `N(μ,σ) clamp [min,max]`
2. **关键公式**：关键结果由 ≥3 个因子决定（参考 design-toolkit.md "Multi-Factor Formulas"）
3. **系统交互**：该系统如何影响/被影响其他系统？画出连接
4. **边界行为**：极端值（0/max）时发生什么？（特殊事件？游戏结束？惩罚？）
5. **阶段变化**：该系统在不同游戏阶段如何演变？（新选项？新约束？）

#### Late-Game Content Escalation

| 阶段 | 事件复杂度 | 决策压力 | 新机制 |
|------|----------|---------|--------|
| 前期 (0-25%) | 单因素事件，2 选项 | 低，容错高 | 核心循环 |
| 发展 (25-50%) | 多因素事件，2-3 选项 | 中，开始有代价 | ≥1 新系统解锁 |
| 成熟 (50-75%) | 复合事件（2+ 问题），3 选项 | 高，明显 trade-off | 系统交互激活 |
| 终局 (75-100%) | 级联事件 + 遗产后果 | 最高，一步影响全局 | 全系统耦合 |

#### Tiered Content Pools

```
earlyPool:   单系统效果，引导性事件（≥10）
midPool:     跨系统效果，中等风险（≥15）
latePool:    复合效果，高风险级联（≥10）
specialPool: 条件解锁，稀有高影响（≥5）
```

#### Phase-Gated Mechanics Examples（模拟经营类）

| 解锁时机 | 新机制 | 设计意图 |
|---------|--------|---------|
| 首次晋升 | 解锁新管理选项（如：招聘高级人才） | 扩展决策空间 |
| 中期里程碑 | 解锁新资源维度（如：社会关系网络） | 引入新策略轴 |
| 后期里程碑 | 已有机制进化（如：基础操作→高级变体） | 防止操作固化 |
| 隐藏条件 | 彩蛋路线（如：特定职业+极端属性=秘密故事线） | 发现储备 |

## AI 叙事冒险类

### 核心循环

```
进入场景 → 阅读叙事 → 做出选择 → 触发后果 → 进入新场景
```

玩家通过选择驱动故事发展，AI 生成动态叙事内容。

### 品质叙事模式

叙事冒险的关键不是"无限自由"，而是**有意义的选择**。

**状态门控场景**：用游戏状态（标记、关系值、物品）控制哪些场景可达。这比纯线性剧情灵活，又比完全开放可控。

```
场景 A ──选择1──→ 场景 B（需要 flag: metNPC）
       ──选择2──→ 场景 C（需要 item: key）
       ──选择3──→ 场景 D（总是可达）
```

### 选择设计

好的选择满足三个条件：
1. **信息不完全**：玩家无法确定哪个选项"最好"
2. **代价明确**：每个选项都有可见的取舍
3. **后果持久**：选择的影响在后续场景中体现

| 选择类型 | 示例 | 设计要点 |
|---------|------|---------|
| 道德两难 | 救一个人还是救多数人 | 没有"正确答案"，两边都有代价 |
| 资源权衡 | 花时间调查 vs 立刻行动 | 一个选项换信息，另一个换时间 |
| 角色定义 | 善意谎言 vs 残酷真相 | 选择塑造玩家角色的性格 |
| 风险赌博 | 安全路线 vs 危险捷径 | 高风险高回报 vs 低风险低回报 |

**避免**：假选择（选项看起来不同但结果一样）、信息不对称选择（一个选项明显更好）。

### 叙事节奏

**场景-续场模式**：

| 节拍 | 作用 | 时长 |
|------|------|------|
| 钩子 | 开场悬念，立刻抓住注意力 | 1 个场景 |
| 探索 | 了解世界、收集信息、建立关系 | 3-5 个场景 |
| 转折 | 打破预期，提升紧张感 | 1 个场景 |
| 升级 | 风险加大，选择更艰难 | 3-5 个场景 |
| 高潮 | 最关键的选择，决定结局走向 | 1-2 个场景 |
| 结局 | 选择的后果完整呈现 | 1 个场景 |

**契诃夫之枪**：前期场景中出现的元素（NPC、物品、信息）必须在后期发挥作用。不要引入无用的元素。

### NPC 设计

每个 NPC 需要：
- **动机**：他想要什么？（与玩家目标可能冲突）
- **秘密**：他隐藏了什么？（驱动剧情发展）
- **态度维度**：对玩家的态度如何变化？（关系系统）

NPC 数量建议：核心 NPC 3-5 个，配角 NPC 不超过核心的 2 倍。太多 NPC 会稀释每个角色的深度。

### 常见陷阱

| 陷阱 | 症状 | 处方 |
|------|------|------|
| 测评表退化 | 游戏变成重复选择题：读描述→选选项→看数值→下一题 | 核心循环必须包含 ≥2 种交互形式（选择题+对话/探索/资源分配等），选择后果必须影响后续场景可达性而非仅改变数值。详见 [assessment-trap.md](blueprint/assessment-trap.md) |
| 选择幻觉 | 玩家发现选什么都一样 | 确保每个选择至少改变一个状态标记 |
| 叙事断裂 | AI 生成内容与前文矛盾 | 用 Message 节点的 historyKey 维护上下文，PromptBuild 的 field fragment 注入关键状态 |
| 节奏拖沓 | 大段文字没有互动 | 每 2-3 段叙事必须有一个选择点 |
| 角色扁平 | NPC 只是信息传递工具 | 给每个 NPC 独立动机和与玩家的冲突点 |
| 元素悬空 | 设计文档列了 NPC/对话/关系系统但核心循环里没有它们 | 核心循环描述中必须包含这些元素作为步骤，否则删掉或降级为装饰性描述 |
| 随机生成器 | AI 产出多样但无意义的内容 | AI 必须达到 3 级+（记忆与连续性），加入回调和叙事穿针 |

### 灵魂要点（AI 叙事类）

AI 叙事最容易陷入"小说"或"随机生成器"反模式。关键对策：
- **叙事穿针**：每回合叙事包含三部分——回响（上回合余波）→ 当下（本回合事件）→ 暗流（下回合暗示）
- **NPC 声线进化**：NPC 说话方式随关系深度变化（陌生→熟悉→信任→亲密），不是静态的
- **不可逆分叉**：在 25% 和 50% 处各设一个永久改变游戏的选择，两条路各有独占内容
- **欲望路径**：对话中除预设选项外允许自由输入，AI 处理意外行为

详见 [soul-design-guide.md](soul-design-guide.md) 支柱 2（AI 原生魔法）和支柱 4（玩家身份）。

### KAL 实现映射

```
Session Prompt/Choice → 场景交互（玩家输入/选择）
PromptBuild + Message → 组装叙事 prompt（含 history、summary、state 数据）
GenerateText          → AI 动态叙事生成
SubFlow               → 叙事子流程复用（narrate.json）
history array         → 叙事上下文维护（GenerateText 自动管理）
Message summaryKey    → 历史压缩摘要注入
Session Branch        → 章节/幕进度判定 + 结局触发
```

### Depth Scaling（Deep tier 专用）

#### Late-Game Narrative Freshness

| 技术 | 描述 |
|------|------|
| 叙事升级 | 后期场景更长、更复杂、情感强度更高 |
| 选择后果加速 | 后期选择的后果更快、更剧烈地体现 |
| NPC 弧线转折 | NPC 在后期展现隐藏面或经历转变 |
| 信息逆转 | 前期的"真相"在后期被颠覆 |
| 分支收束 | 前期不同路径在后期汇聚到不同的高潮场景 |

#### Tiered Scene Pools

```
Act 1:    探索性场景，建立世界观（≥8 场景）
Act 2:    冲突升级场景，加深关系（≥10 场景）
Act 3:    高潮场景，重大选择（≥5 场景）
Secret:   条件解锁场景（≥3 场景）
```

## 对话冒险类

### 核心循环

```
遇见角色 → 进入对话 → 选择回复 → 关系变化 → 触发新内容
```

以对话为核心交互方式，通过与 NPC 的对话推进游戏。

### 对话系统设计

**对话结构**：

| 层级 | 说明 | 示例 |
|------|------|------|
| 对话轮 | 单次 NPC 发言 + 玩家回复 | "你好" → 选择回复 |
| 对话段 | 围绕一个话题的多轮对话 | 讨论一个任务（3-8 轮） |
| 对话场景 | 一次完整的会面 | 在酒馆遇见 NPC（1-3 段） |

**每轮对话的回复选项**：2-4 个，风格差异明显（友善/中立/挑衅/特殊）。特殊选项由游戏状态解锁。

### 角色关系网络

设计要点：
1. **关系维度**：不只是"好感度"单一数值。可以有信任、尊重、亲密等多维度
2. **关系影响**：关系值影响 NPC 的回复态度、可用对话选项、剧情分支
3. **三角关系**：至少设计一组 NPC 之间的关系张力（提升 A 的好感可能降低 B 的好感）

### 常见陷阱

| 陷阱 | 症状 | 处方 |
|------|------|------|
| 对话疲劳 | 玩家跳过对话内容 | 缩短每段对话长度，增加选择频率 |
| 讨好循环 | 总是选最友善的选项 | 让"友善"选项有代价（如暴露弱点） |
| 关系无感 | 好感度变化但玩家不在意 | 让关系值直接影响可见的游戏内容 |

### 灵魂要点（对话冒险类）

对话冒险的灵魂在于让 NPC 感觉"真实"。关键对策：
- **声线进化**：NPC 说话方式随关系阶段自然变化，不要暴露"关系等级"这个概念
- **玩家 DNA 追踪**：追踪玩家的对话风格（总是安慰？总是挑衅？），让 NPC 回应这个模式
- **身份定义时刻**：设计至少一个必须在两个深度关系之间选择的时刻

详见 [soul-design-guide.md](soul-design-guide.md) 支柱 2（AI 原生魔法）中的 NPC 声线进化模式。

### KAL 实现映射

```
Session Prompt        → 多轮对话入口（玩家输入传入 Flow）
PromptBuild           → NPC 人格 prompt（含关系历史、性格、语调规则）
Message               → 注入对话历史（historyKey 按 NPC 分离）
GenerateText          → NPC 回复生成
WriteState            → 关系值/好感度回写
State keys            → NPC 数据（关系值、对话摘要、性格标签）
Session Branch        → 章节进度判定
```

### Depth Scaling（Deep tier 专用）

#### Relationship Depth Escalation

| 关系阶段 | 对话深度 | 解锁内容 |
|---------|---------|---------|
| 陌生 (0-25) | 表面寒暄，2 选项 | 基本信息 |
| 熟悉 (25-50) | 观点交流，2-3 选项 | 个人故事 |
| 信任 (50-75) | 深度对话，3 选项含特殊选项 | 秘密、弱点 |
| 亲密 (75-100) | 情感对话，解锁独占内容 | 剧情转折、专属结局 |

#### Late-Game Dialogue Freshness

| 技术 | 描述 |
|------|------|
| 角色弧线 | NPC 性格/态度随关系深入而演变，不是静态好/坏 |
| 关系张力网 | 提升 A 的好感可能损害 B 的关系（三角张力） |
| 秘密揭露 | 高信任解锁 NPC 隐藏背景，改变剧情走向 |
| 背叛/考验 | 后期关系面临重大考验，选择决定关系最终走向 |
