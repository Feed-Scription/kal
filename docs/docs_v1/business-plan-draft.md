## 一、一句话描述

**KAL-AI 是 AI 原生游戏的 "Unity"——一个 TypeScript 开源框架 + 分发平台，让开发者用 10 倍速构建以 AI 为核心玩法的游戏。**

## 二、我们要解决的问题

### 2.1 行业痛点

AI 原生游戏（以 LLM 为核心机制的游戏）正在中国市场爆发。2025-2026 年初，《青椒模拟器》（DAU 9 万，累计 60 万用户）、《大厂模拟器》（首日 PV 5 万+）、《历史模拟器：崇祯》（Steam 首测）等爆款密集出现。小红书上已形成完整的 AI 文游创作和交易生态。

但所有开发者都面临相同的困境：

| 痛点 | 表现 | 引用 |
|------|------|------|
| **没有标准化框架** | 每个人从零搭建后端 API 调度 | 青椒开发者："设计了几个后端请求 API 接口，每个接口对应不同模型" |
| **响应延迟 10-30 秒** | 玩家等待时间过长，体验差 | 青椒开发者："AI 做游戏最大难点是响应慢" |
| **Token 成本失控** | 小团队自掏数万元维持运营 | 青椒上线后开发者自掏腰包数万元买 Token |
| **AI 幻觉破坏体验** | 世界观不一致、时间线错乱 | 崇祯在 1627 年出现 1644 年才有的"大顺" |
| **完全没有测试手段** | 上线靠运气，质量靠手感 | 所有案例中无一提及系统性测试 |

**核心洞察：这些开发者有 AI 能力、有创意想法，但缺乏游戏工程基础设施。他们正在用 50% 的时间造轮子。**

### 4.2 核心框架能力

**@kal-ai/core**（开源，TypeScript，浏览器 + Node.js 双端）

| 模块 | 解决的问题 | 替代方案 |
|------|-----------|---------|
| **ModelFactory** | 多模型注册 + 按需调度 + 中间件 | 手写 if/else 选模型 |
| **Flow** | DAG 声明式编排，自动拓扑排序 + 并行 | 手写 async 调用链 |
| **StateManager** | 游戏世界状态管理 + 写冲突检测 | 全局变量或 Redux |
| **Prompt** | 函数式 Prompt 片段组合 | 字符串拼接 |
| **CacheManager** | 三层缓存（L1 内存 / L2 本地 / L3 远程） | 无缓存或手写 Map |
| **SafetyFilter** | 内容安全检测 + AI 幻觉缓解 | 无或手写正则 |
| **JsonRepair** | LLM 输出 JSON 格式修复 | try/catch 重试 |
| **Observer** | Token 用量追踪 + 计费统计 | 手写日志或无追踪 |

#### 成本侧：真实 API 请求日志（2026.02.28）

以下为 FeedScription 上一款典型游戏的实时 API 请求日志，所有请求均使用 `xiaomi/mimo-v2-flash`：

| Feature 类型 | 用途 | 平均 Tokens | 平均 Credits | 平均延迟 | 成本（¥） |
|-------------|------|------------|-------------|---------|----------|
| `quarter` | 季度推进/结算 | 6,050 | 1.14 cr | 7,900ms | ¥0.0114 |
| `event` | 随机事件生成 | 5,390 | 1.03 cr | 7,600ms | ¥0.0103 |
| `sales` | 经费/推销事件 | 3,950 | 0.68 cr | 4,800ms | ¥0.0068 |
| `chat` | 对话交互 | 3,870 | 0.67 cr | 3,900ms | ¥0.0067 |
| `social_post` | 朋友圈生成 | 3,510 | 0.56 cr | 3,000ms | ¥0.0056 |

**关键发现：一次玩家"回合"并非一次 API 调用，而是一条多步管线。**

玩家点击"推进一个季度"，背后触发的调用链为：

```
玩家操作："推进季度"
  │
  ├── quarter（季度结算）      1.14 cr   7.9 秒
  ├── event × 1-2（随机事件）   1.03 cr   7.6 秒  ← 可并行
  ├── social_post × 2-3（朋友圈）0.56 cr   3.0 秒  ← 可并行
  ├── chat（如有对话交互）      0.67 cr   3.9 秒
  └── sales（如有经费事件）     0.68 cr   4.8 秒
  ─────────────────────────────────────────────────
  单回合总计：约 4-7 次调用 ≈ 3.5-5.5 cr ≈ ¥0.035-0.055
  总延迟（串行）：约 25-35 秒
  总延迟（并行）：约 8-12 秒 ← KAL-AI Flow 的价值
```

**这直接证明了 KAL-AI Flow DAG 编排的必要性：**
- `social_post × 3` 之间互不依赖，可以**并行**
- `event` 和 `social_post` 可以在 `quarter` 完成后**并行触发**
- 串行执行需要 25-35 秒；合理并行后可压缩到 8-12 秒
- KAL-AI 的三级优先级让 `chat`（玩家正在等待）优先于 `social_post`（后台生成）

```
每次 API 调用按实际 token 消耗 × 2 倍（50% 毛利）定价。

实际示例：
```

#### 关键数据洞察

1. **38.2% 活跃→付费转化率是核心壁垒**：F2P 游戏行业均值为 1.5-3%（来源：OnRanko Digital Stats / Moloco 2025 Mobile Gaming Trends），微信小游戏 IAP 转化约为 APP 手游的 2-3 倍（来源：2026 微信小游戏广告增长白皮书），即 ~3-9%。FeedScription 38.2% 是行业均值的 **4-25 倍**。原因：
    - **商业模式定性**：这不是传统的"道具内购"，而是 **"街机投币模式" (Play-to-Pay) 或类 SaaS 订阅**。AI 消耗是刚性的，不付费无法继续体验核心内容，因此转化逻辑更接近工具软件。
    - **其他因素**：(a) 300 cr 免费试玩让用户先体验完整价值；(b) AI 文游沉浸感强，一旦上手就想继续；(c) ¥10-50 的甜区定价精准命中目标用户承受力；(d) **Day 0 即转化**——不给用户犹豫期。
2. **61.4% 付费用户 D1 留存 ≈ 全行业 Top 1%**：GameAnalytics 2026 Mobile Benchmarks 数据显示，D1 留存 Top 1% 为 64-68%，Top 10% 为 ~40%，中位数已降至 <1%。FeedScription 付费用户 61.4% 的 D1 留存**接近全行业最顶尖水平**
3. **LTV/CAC = 2.47**：接近健康阈值（>3 为优秀）。提升路径明确——增加游戏品类提升复购率（22.8% → 35%+），LTV 将从 ¥35.60 升至 ¥50+
4. **94% 平台分润率**：核心收入来自 AI 推理中间差价，不是游戏分销抽成。这是一个**类 infra 的高毛利模型**
5. **青椒模拟器 = 70% 收入**：单品依赖度高，急需更多爆款。这正是 KAL-AI 框架降低开发门槛的战略意义——每多一款好游戏，平台风险就降低一分
6. **仅 6 款游戏、4 个开发者**就做到这些指标：供给侧是当前唯一瓶颈
